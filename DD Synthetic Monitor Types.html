DataDog Synthetics SOP

Standard Operating Procedure

I. PURPOSE

The purpose of this process is to:

  • Continuously monitor the end-user experience to detect website performance issues.
  • Find and fix web performance issues in both internal and customer-facing applications before users are affected.
  • Check if web servers or applications are up, down, or experiencing high loads.

II. SCOPE

This process applies to all members of the Command Center.

III. PROCESS

This process applies to Provider, Avalere, ComplexCare Solutions, Pharmacy and Payer Synthetic Monitors recorded in Datadog that simulate real user activity on a website. The automated user activity such as logging into the site or selecting various links on the page are played at various intervals from various locations to detect website performance issues. The steps to manually validate the websites are documented in this SOP.

Synthetic Monitor Types: TX & API's

Synthetic TX (Browser) Monitor

Synthetic Transactions (TX) are recorded steps that simulate real user activity on a website such as logging into the site or selecting various links on the page. The recorded activity is played at various intervals from various locations to detect website performance issues. The TX and steps are documented in this SOP.

Synthetic API (HTTPS - SSL - ICMP - DNS) Monitor

API monitors test endpoints and proactively monitor that important services are available at any time and from anywhere. API tests come in different types that allow launching of requests on the different network layers of the systems.

API Monitor Types

  • HTTPS Test – Synthetic API (HTTPS) Monitor – Synthetic HTTP API’s test a web server’s ability to accept incoming sessions over a secure channel and then transmit the requested page; allows sending of HTTP requests to our application API endpoints in order to verify that they are responding to the requests, as well as that the site meets any conditions defined ie: response time, status code and header or body content.
  • SSL Test – Synthetic API (SSL) Monitor – Synthetic SSL API’s allow proactive monitoring of the validity and expiration of your SSL/TLS certificates to ensure secure connections between our key services and user. Be sure to escalate these at 21 and 7 Days to expiry. An alert will be received at 21 days; monitor the CC Dashboard to determine Day 7. See entry #1 in the SYNTHETIC MONITOR INFORMATION CHART for further details on SSL escalations.
  • ICMP Test – Synthetic API (ICMP) Monitor – Synthetic ICMP API’s allow you to easily monitor the availability of your hosts and diagnose network communication issues.
  • TCP Test–CC is not monitoring with this type of API at this time.
  • DNS Test - CC is not monitoring with this type of API at this time.
